# Python SDK

The GlassFlow Python SDK provides a programmatic way to create and manage data pipelines. This approach is ideal for developers who prefer code-based configuration, automated deployment, and integration with existing Python workflows.

## Installation

Install the GlassFlow Python SDK using pip:

```bash
pip install glassflow>=3.6.0
```

> **Pipeline version notice**: The default pipeline version is now `v3`. If you have existing `v2` configurations, see [Migrating from V2 to V3](#migrating-from-v2-to-v3) below. Pipeline version `v2` is deprecated and will be removed in a future release.

## Basic Usage

1. **Import Required Modules**

```python
from glassflow.etl import Client
```

2. **Create a Client**

```python
client = Client(host="http://localhost:30180")
```

**Note**: The `host` parameter specifies the GlassFlow API host (if GlassFlow is running locally with [GlassFlow CLI](/installation/cli), it is `http://localhost:30180`).

3. **Create a Pipeline from JSON Config**

Create a pipeline from a JSON pipeline configuration file:
```python
pipeline = client.create_pipeline(pipeline_config_json_path="pipeline_config.json")
```

## Pipeline Management

### Get a Pipeline

```python
pipeline = client.get_pipeline(pipeline_id="<my-pipeline-id>")
```

### Get Pipeline Health

Access the pipeline health status:
```python
pipeline = client.get_pipeline(pipeline_id="<my-pipeline-id>")
print(pipeline.status)
```

Output:
```
<PipelineStatus.RUNNING: 'Running'>
```

or query the latest health status:

```python
pipeline_health = pipeline.health()
print(pipeline_health)
```

Output:
```json
{
    "pipeline_id": "<pipeline-id>", 
    "pipeline_name": "<pipeline-name>", 
    "overall_status": "Running", 
    "created_at": "2025-09-05T10:21:57.945135078Z", 
    "updated_at": "2025-09-05T10:21:58.192448618Z"
}
```

### List Pipelines

You can list all pipelines available in your GlassFlow deployment:
```python
pipelines = client.list_pipelines()
print(pipelines)
```

Output:
```json
[
    {
        "pipeline_id": "<my-pipeline-id>", 
        "name": "<my-pipeline-name>", 
        "transformation_type": "Deduplication", 
        "created_at": "2025-09-04T15:20:24.327977208Z", 
        "updated_at": "2025-09-04T15:20:24.327977208Z",
        "status": "Running",
        "metadata": {
            "tags": ["demo"]
        }
    }
]
```

### Stop a Pipeline

Stopping a pipeline will stop ingesting new messages and once all messages in the queue are processed, the pipeline will stop.

```python
pipeline.stop()
```

### Terminate a Pipeline

Terminating a pipeline will stop ingesting new messages and immediately terminate the pipeline.

```python
pipeline.stop(terminate=True)
```

### Resume a Pipeline

Resume will bring the pipeline back to running state from stopped state.

```python
pipeline.resume()
```

### Delete a Pipeline

Only stopped (or terminated) pipelines can be deleted.

```python
pipeline.delete()
```

or 

```python
client.delete_pipeline(pipeline_id="<my-pipeline-id>")
```

### Update a Pipeline

Only stopped (or terminated) pipelines can be updated.

```python
pipeline.update(config_patch=<pipeline config patch>)
```

Example:

```python
pipeline.update(config_patch={"join": {"type": "join", "enabled": true}})
```

or update the full pipeline config:

```python
pipeline.update(config_patch=<new pipeline config>)
```

## Dead-Letter Queue (DLQ)

### Get DLQ State

```python
dlq_state = pipeline.dlq.state()
print(dlq_state)
```


### Purge DLQ

```python
pipeline.dlq.purge()
```

Output:
```json
{
    "last_consumed_at": "2025-09-05T10:32:38.113427621Z",
    "last_received_at": "2025-09-05T10:23:26.530466989Z",
    "total_messages": 2,
    "unconsumed_messages": 1
}
```

### Read Messages from DLQ

```python
messages = pipeline.dlq.consume(batch_size=1)
print(messages)
```

Output:
```json
[
    {
        "component": "ingestor",
        "error": "failed to validate data",
        "original_message": "<original kafka message>"
    }
]
```

## Import / Export Pipeline Configurations

### Import

Import a pipeline configuration from a JSON file:
```python
pipeline_config = client.create_pipeline(pipeline_config_json_path="<my-pipeline-config.json>")
```

Import a pipeline configuration from a YAML file:
```python
pipeline_config = client.create_pipeline(pipeline_config_yaml_path="<my-pipeline-config.yaml>")
```

### Export

Export a pipeline configuration to a JSON file:
```python
pipeline.to_json(json_path="<my-pipeline-config.json>")
```

Export a pipeline configuration to a YAML file:

```python
pipeline.to_yaml(yaml_path="<my-pipeline-config.yaml>")
```

## OTLP Source Pipelines

GlassFlow can ingest OpenTelemetry signals directly without a Kafka cluster. Set `source.type` to `otlp.logs`, `otlp.metrics`, or `otlp.traces`. GlassFlow manages the OTLP receiver endpoint — you configure only the sink and schema.

Deduplication is supported at the source level. Join is not supported for OTLP source pipelines.

```python
pipeline_config = {
    "version": "v3",
    "pipeline_id": "otel-logs-pipeline",
    "source": {
        "type": "otlp.logs",
        "deduplication": {
            "enabled": True,
            "key": "trace_id",
            "time_window": "5m"
        }
    },
    "join": {"enabled": False},
    "sink": {
        "type": "clickhouse",
        "host": "clickhouse.example.com",
        "port": "9000",
        "database": "otel",
        "username": "default",
        "password": "mysecret",
        "secure": True,
        "table": "otel_logs"
    },
    "schema": {
        "fields": [
            {"source_id": "otlp.logs", "name": "timestamp",     "type": "string", "column_name": "timestamp",     "column_type": "DateTime64(9)"},
            {"source_id": "otlp.logs", "name": "severity_text", "type": "string", "column_name": "severity_text", "column_type": "LowCardinality(String)"},
            {"source_id": "otlp.logs", "name": "body",          "type": "string", "column_name": "body",          "column_type": "String"},
            {"source_id": "otlp.logs", "name": "trace_id",      "type": "string", "column_name": "trace_id",      "column_type": "String"},
            {"source_id": "otlp.logs", "name": "service_name",  "type": "string", "column_name": "service_name",  "column_type": "LowCardinality(String)"}
        ]
    }
}

pipeline = client.create_pipeline(pipeline_config)
```

Available signal types and their common fields:

| `source_type`   | Key fields available for filter/transform/mapping |
|-----------------|--------------------------------------------------|
| `otlp.logs`     | `timestamp`, `severity_text`, `severity_number`, `body`, `trace_id`, `span_id`, `service_name`, `attributes.*` |
| `otlp.metrics`  | `name`, `description`, `unit`, `timestamp`, `data_point_value`, `attributes.*` |
| `otlp.traces`   | `trace_id`, `span_id`, `parent_span_id`, `name`, `start_time`, `end_time`, `status_code`, `attributes.*` |

## Pipeline Resources

Use `pipeline_resources` to set CPU, memory, replica counts, and storage limits per component. This is optional — omit it to use GlassFlow's defaults.

```python
pipeline_config = {
    "version": "v3",
    "pipeline_id": "my-pipeline-id",
    # ... source, join, sink, schema ...
    "pipeline_resources": {
        "ingestor": {
            # Single-source pipeline: use "base"
            "base": {
                "replicas": 2,
                "requests": {"cpu": "250m", "memory": "256Mi"},
                "limits":   {"cpu": "500m", "memory": "512Mi"}
            }
            # Join pipeline: use "left" and "right" instead of "base"
        },
        "transform": {
            "replicas": 1,
            "requests": {"cpu": "500m", "memory": "512Mi"},
            "limits":   {"cpu": "1000m", "memory": "1Gi"},
            "storage":  {"size": "10Gi"}
        },
        "sink": {
            "replicas": 1,
            "requests": {"cpu": "250m", "memory": "256Mi"},
            "limits":   {"cpu": "500m", "memory": "512Mi"}
        },
        "nats": {
            "stream": {
                "maxAge":   "24h",
                "maxBytes": "10Gi"
            }
        }
    }
}
```

The following resource fields are **immutable** — they must be set at creation time and cannot be changed by `pipeline.update()`:

| Field | Reason |
|-------|--------|
| `transform.storage.size` | Underlying volume cannot be resized |
| `join.replicas` | Join topology is fixed after creation |
| `nats.stream.maxAge` | JetStream stream config is immutable |
| `nats.stream.maxBytes` | JetStream stream config is immutable |

## Migrating from V2 to V3

Pipeline version `v2` is deprecated. Update your configuration dictionary as follows before passing it to `client.create_pipeline()`.

| Area | V2 field | V3 field | Notes |
|------|----------|----------|-------|
| `version` | `"v2"` | `"v3"` | Required change |
| Topics | _(no `id`)_ | `"id": "<topic-name>"` | Each topic now requires an explicit `id` |
| Deduplication | `"id_field": "field_name"` | `"key": "field_name"` | `id_field` is removed in V3 |
| Deduplication | `"id_field_type": "string"` | _(removed)_ | Type is inferred from the schema |
| Join sources | `"join_key": "field_name"` | `"key": "field_name"` | `join_key` is removed in V3 |
| Join sources | _(no `fields`)_ | `"fields": ["f1", "f2"]` | Optional — selects output fields |
| Sink password | base64-encoded string | plain text string | Passwords are no longer base64-encoded |
| Sink | _(no `http_port`)_ | `"http_port": "8123"` | Optional HTTP port for ClickHouse |

**Before (V2):**

```python
{
    "version": "v2",
    "source": {
        "topics": [{
            "name": "users",
            "deduplication": {
                "enabled": True,
                "id_field": "event_id",
                "id_field_type": "string",
                "time_window": "1h"
            }
        }]
    },
    "join": {
        "enabled": True,
        "sources": [{"source_id": "users", "join_key": "user_id", "time_window": "10m", "orientation": "left"}]
    },
    "sink": {"password": "c2VjcmV0", ...}
}
```

**After (V3):**

```python
{
    "version": "v3",
    "source": {
        "topics": [{
            "id": "users",
            "name": "users",
            "deduplication": {
                "enabled": True,
                "key": "event_id",
                "time_window": "1h"
            }
        }]
    },
    "join": {
        "enabled": True,
        "sources": [{"source_id": "users", "key": "user_id", "time_window": "10m", "orientation": "left", "fields": ["user_id", "event_id"]}]
    },
    "sink": {"password": "mysecret", ...}
}
```
