# GlassFlow ClickHouse ETL UI — Design Context

## Design Context

### Users
Data engineers configuring Kafka→ClickHouse ETL pipelines. Extended technical work sessions, primarily desktop. Comfortable with infra tooling; prioritize clarity over delight. They know what they want but need the tool to help them move fast without making mistakes.

### Brand Personality
Modern premium dev tool — **Precise, Energized, Sophisticated**. Orange as warmth+energy in a cold enterprise space. Not playful, not enterprise-gray, not terminal-stark.

### Aesthetic Direction
Dark-mode native. Near-monochromatic charcoal (#0e0e10 to #161617 elevation stack) with a single amber-orange accent (#ffa24b). Gradient CTA button is a signature brand moment. Premium, focused, technical credibility with visual quality. Anti-references: Oracle/SAP gray monotony, playful rounded SaaS, stark black terminals.

### What to preserve
- Primary color: `--color-orange-300` (#ffa24b) and its gradient (#feac5e → #e28024)
- Dark theme (dark-only, no light mode)
- The gradient CTA button
- The nav tab underline gradient indicator
- The `surface-gradient-border` modal signature

### Design Principles
1. **Data engineers first** — clarity over decoration; explicit over clever
2. **Premium, not generic** — fight the admin-template look; differentiate through intentional composition
3. **Precision in details** — spacing, typography, states must be right; technical users notice sloppiness
4. **Progressive disclosure** — complex configurations revealed step-by-step, not all at once
5. **Trust through stability** — no flashy animations on data-critical moments; reserve motion for state communication

### Token system
Full token system in `src/themes/base.css` (primitives) + `src/themes/theme.css` (semantic). Never use raw Tailwind color utilities for semantic/brand colors. All spacing via `--unit-x*` tokens (4px base). Typography via `.title-*`, `.body-*`, `.caption-*` utility classes.
