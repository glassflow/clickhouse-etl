What lives above the pipeline table?

The command center needs a header section. The question is how much fleet-level aggregation you want before the per-pipeline rows.

A

Stat cards only (status signals)

4 KPI cards: Running / Needs attention / Paused / DLQ backlog. These come from the pipeline list API — no VM query needed. Fast, always available.

Pros

  • Zero extra VM queries
  • Always current (from pipeline list API)
  • Already built — just needs cleanup

Cons

  • No throughput trend at the fleet level
  • Can't answer "is total system load rising?"
B

Stat cards + fleet-wide throughput chart

Stat cards + a single fleet-wide throughput chart showing total events/s across all pipelines over the selected time range. One VM query with no pipeline_id filter (or sum of all). Shows the system pulse.

Pros

  • Shows "is the system pulse healthy?" at a glance
  • Good for on-call engineers — one chart captures everything
  • Only 1 extra VM query (sum across all pipelines)

Cons

  • Requires VM to be enabled — hidden otherwise
  • More visual complexity in the header
